# ОТЧЁТ по лабораторной работе №6

## Название работы
Разработка и анализ прототипа приложения/сервиса FinOptima с помощью средств вайб-кодинга

## Цель и задачи работы

**Цель:** Разработать и проанализировать прототип информационной системы FinOptima для учета личных финансов и предоставления персонализированных рекомендаций с применением средств вайб-кодинга и больших языковых моделей.

**Задачи:**
1. Проанализировать описание системы FinOptima и определить её принадлежность к классам информационных систем
2. Спроектировать архитектуру веб-сервиса, структуру данных и пользовательские сценарии
3. Реализовать прототип веб-приложения на Django с использованием Cursor и LLM
4. Провести функциональное тестирование прототипа
5. Выполнить анализ качества кода и эффективности применения вайб-кодинга

## Текст анализа результата

В ходе разработки прототипа FinOptima средства вайб-кодинга (Cursor, большие языковые модели) применялись на всех этапах: от проектирования архитектуры до написания кода и подготовки документации.

**Степень успешности разработки:**
Прототип успешно реализует базовый функционал: регистрацию пользователей, добавление транзакций, отображение финансовой статистики и генерацию рекомендаций. Все основные сценарии использования работают корректно после устранения начальных ошибок настройки.

**Качество кода:**
Сгенерированный код соответствует стандартам Django: использованы классовые представления, ORM, шаблонизатор Django Templates. Код структурирован, имеет понятные имена переменных и функций. Однако потребовалась ручная доработка: исправление импортов, настройка URL-маршрутов, добавление декораторов авторизации.

**Производительность:**
Прототип демонстрирует быстрый отклик при работе с небольшими объёмами данных благодаря использованию SQLite и эффективным ORM-запросам. Для продакшена потребуется оптимизация: переход на PostgreSQL, кэширование, асинхронная обработка.

**Возможности развёртывания:**
Приложение готово к развёртыванию на хостингах с поддержкой Python (Heroku, Railway, PythonAnywhere). Необходимо лишь настроить переменные окружения и статические файлы.
